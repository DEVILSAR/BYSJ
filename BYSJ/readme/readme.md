随着计算机技术的飞速发展，数据存储技术也不断进步，从最初的物理存储，发展到如今广泛运用的云存储。云存储克服了传统物理存储的局限性，为用户提供了成本低廉、存储方便、访问快捷的高质量高性能服务。然而，随着用户和数据量的不断增加，云存储服务器空间日益不足。因此，数据去重技术应运而生，并且在云存储中得到了广泛的运用。然而，由于近年来频发的服务器泄密等安全事件，导致用户对其半可信，往往选择加密数据再上传，从而严重影响了云服务器端的去重效率，云存储加密数据安全去重问题亟待解决。本文在近年来相关研究基础上，提出了两种切实可行的云存储加密数据安全去重方案。
1.基于bloom filter的云存储加密安全去重方案
在客户端用户上传数据前，基于离散数学中的对数困难问题，构造对应的去重检测标签，发送至可信的第三方服务器，引入bloom filter即布隆过滤器，提升查询的时间和空间效率，最后加密相应明文和密钥，统一发送给云存储服务器，并创建物理指针，完成云存储加密数据安全去重。与SecRes、CE、MLE、Dupless等方案相比，去重标签查询时间开销大幅降低，且保证了云存储下数据的安全性。
2.基于流行度划分的云存储加密安全去重方案
引入流行度划分概念，基于云存储服务器中文件数据上传或存储次数，设定阈值T，引入流行度检测标签t，若t < T,该数据为非流行数据，即隐私度安全性较高的用户信息、密码等，采用随机参数加密上传至云服务器保存；若t > T,该数据为流行数据，即隐私度安全性较低的公开文本、文档等，采用收敛加密上传保存，最后在服务器端创建物理指针，完成云存储加密数据安全去重过程。与Wang等人、CE等方案相比，在支持非流行数据加密去重的基础上大幅减少了标签查询时间开销，且保证了云存储下数据的安全性。
